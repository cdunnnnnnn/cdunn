Intro

I wrote a pen a few months back experimenting with an idea for a button.

The markup

<a class="btn" data-js="btn">
  <span class="btn-inr">
    <span class="txt-a">See this button?</span>
    <span class="txt-b">Now ya do.</span>
  </span>
</a>

The styles

&:after {
  content: "";
  position: absolute;
  top: 0;
  left: 0;
  display: block;
  width: 100%;
  height: 100%;
  background: linear-gradient(135deg, #2980b9 0%,#9b59b6 100%);
  background-size: 100% auto;
  opacity: 0;
  transition: all .65s ease-in-out;
}

The scripts

Looking at the js, there’s a simple function that adds a blur class to the fullscreen gradient when the mouseenter event fires. This class is styled with a semi-transparent opacity and transitions in when a the button is hovered. Simultaneously the script also adds an active class to the button, which essentially swaps out the text elements. Once the mouseleave event fires, both classes are removed to return the elements back to their normal state. Theoretically, you could call this function on any event you want: click, drag, focus, scroll, etc…

// vanilla js
function toggleButton(el) {
  var body = document.body;
  var element = document.querySelector(el);

  element
    .addEventListener('mouseenter', function(e) {
      e.target.classList.add('active');
      body.classList.add('blur');
    }, false)
  ;

  element
    .addEventListener('mouseleave', function(e) {
      e.target.classList.remove('active');
      body.classList.remove('blur');
    }, false)
  ;
};

…And you can do the same thing if you’re using the jQuery:

// jquery
$('[data-js="btn"]')
  .hover(function(){
    $(this).toggleClass('active');
    if ($(this).hasClass('active')) {
      $('body').addClass('blur');
    } else {
      $('body').removeClass('blur');
    }
  })
;
